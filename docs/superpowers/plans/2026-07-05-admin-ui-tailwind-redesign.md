# Admin UI Tailwind Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Redesign the entire private admin UI into a clean, modern, responsive dashboard using Tailwind CSS while preserving all existing admin workflows.

**Architecture:** Keep the existing Express + EJS server-rendered architecture and introduce a small Tailwind build pipeline that emits a static `public/styles/admin.css` file. Convert the shared layout into an app shell and update each EJS screen in place so routes, controllers, forms, and database behavior remain unchanged. Add integration tests that verify CSS is served and key admin screens keep their functional links, forms, status labels, and copyable endpoint data.

**Tech Stack:** Express 4, EJS 6, Node test runner, better-sqlite3, Tailwind CSS 4.3.2, @tailwindcss/cli 4.3.2, @tailwindcss/forms 0.5.11, supertest 7.2.2, cheerio 1.2.0.

## Global Constraints

- Do not change public API routes, admin route URLs, form field names, model signatures, or database schema.
- The private admin app remains served on `ADMIN_PORT`, currently documented as `8667`.
- Use Tailwind CSS classes and a compiled static stylesheet at `/assets/styles/admin.css`.
- Keep the UI operational without client-side JavaScript except existing browser confirmation for destructive deletes.
- Do not introduce a frontend SPA, bundler, hydration framework, or component runtime.
- Use a restrained operational-tool visual style: dense, scannable, modern, and clean.
- Keep cards at `8px` radius or less by using Tailwind `rounded-lg` only for panels, table containers, and message bubbles.
- Ensure mobile layouts do not overflow horizontally except intentionally scrollable data tables.

---

## File Structure

- Modify `package.json`: add Tailwind build scripts and admin UI test dependencies.
- Modify `package-lock.json`: updated automatically by `npm install`.
- Create `assets/styles/admin.css`: Tailwind v4 source, theme tokens, reusable component classes via `@layer components`.
- Create `public/styles/admin.css`: compiled Tailwind output generated by `npm run build:css`.
- Modify `server.js`: serve `public/` under `/assets` for the private admin app.
- Modify `views/layout-head.ejs`: replace inline CSS with the dashboard shell, stylesheet link, header, navigation, and page background.
- Modify `views/layout-foot.ejs`: close the new shell markup.
- Modify `views/projects/list.ejs`: redesign the project index with stats, empty state, responsive table, and polished actions.
- Modify `views/projects/form.ejs`: redesign create/edit project form, endpoint callout, repos section, and API groups section.
- Modify `views/conversations/list.ejs`: redesign conversations and latest API calls as dashboard tables with badges.
- Modify `views/conversations/detail.ejs`: redesign message transcript as a readable timeline.
- Create `tests/adminUi.test.js`: integration coverage for layout assets and the redesigned admin workflows.
- Modify `README.md`: document `npm run build:css` and `npm run dev:css` for local UI work.
